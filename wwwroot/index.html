Form Demo — CS-366 Week 7

Each card below contains an HTML form. Submit each one and observe: the URL bar, the server response, and the DevTools Network tab (F12).

There are no controllers, no models, no views. The server reads raw query strings and form bodies — exactly what MVC does behind the scenes.

1. Simple Search GET

Type something and submit. Watch the URL bar — your search term appears as ?search=...

2. Multiple Fields (with a bug!) GET

Fill in the fields and submit. Look at the URL — notice anything weird about "Ending Date"?

3. Login (WRONG way) GET

Don't do this! Submit and look at the URL bar — your password is visible to everyone.

4. Login (RIGHT way) POST

Same fields, but using POST. Submit and check the URL bar — no password! Open DevTools > Network > Payload to find it.

5. Create with File Upload POST

This form uses enctype="multipart/form-data" so the browser sends file contents (not just the filename). Check the Content-Type in DevTools — it's multipart/form-data instead of application/x-www-form-urlencoded.
